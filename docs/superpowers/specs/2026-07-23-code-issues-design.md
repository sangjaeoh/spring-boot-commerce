# 코드 이슈 4건·인증 표면표 압축 설계

## 배경

문서 전수 동기화 감사가 문서와 무관한 코드 이슈 4건을 발견했다. 포트폴리오 목표상 리뷰어가 발견 가능한 실결함이라 수정한다. 함께 문서 후속 과제 1건(인증 표면표 압축)을 처리한다.

## 항목

### 1. inquiry 작성 자격 게이트·상품 검증

- 현상: 문의 작성에 자격 게이트가 없어 정지·탈퇴 회원도 작성 가능하고, 존재하지 않는 상품에도 문의가 달린다.
- 수정: 작성 시 자격 활성(활성 ∧ 미탈퇴) 회원 검증과 상품 존재·미삭제 검증을 추가한다. 검증은 앱 계층 파사드가 MemberReader·ProductReader로 조율한다(도메인 간 컴파일 의존 없음 유지). 노출 여부는 검증하지 않는다 — 구매 후 숨김된 상품에도 문의할 수 있어야 한다.
- 문서: REQUIREMENTS·DOMAIN_MODEL 문의 절에 자격·상품 검증 서술을 추가한다.

### 2. review 삭제 소프트삭제 통일

- 현상: 본인 삭제는 물리 DELETE(재작성 가능), 관리자 삭제는 소프트삭제(재작성 불가)로 비대칭이다.
- 수정: 본인 삭제를 소프트삭제로 전환하고 기존 removedReason 계열 필드를 재사용한다. 본인 삭제 후 재작성은 허용을 유지한다 — 구매당 1회 유니크를 미삭제 행에만 적용한다(partial unique index, 마이그레이션 추가). 관리자 삭제 후 재작성 불가는 현행 유지한다(제재 성격).
- 문서: REQUIREMENTS·DOMAIN_MODEL 리뷰 절의 삭제 서술을 갱신한다.

### 3. 재입고 알림 발송 단위 추적

- 현상: 알림 발송이 부분 실패하면 다음 시도에서 성공분 포함 전원 재발송된다(중복 알림).
- 수정: 행 단위 발송 성공 마킹으로 전환한다 — 발송 성공한 알림 신청 행만 마킹·커밋하고, 실패분만 재시도 대상으로 남긴다.
- 문서: DOMAIN_MODEL 위시리스트 절의 발송 서술을 갱신한다.

### 4. 부분환불 PG 취소 거래 영속 (PaymentRefund 신설)

- 현상: `DefaultPaymentProcessor.cancelPartially`가 PG 부분취소 반환값(취소 거래 ID)을 버린다. Payment에는 누계(`refundedAmount`)만 남아 부분환불 건별 추적이 불가능하다 — REQUIREMENTS "승인·취소 거래를 각각 식별·추적" 서술이 부분환불에서 성립하지 않는다.
- 수정: `payment_refund` 테이블·PaymentRefund 엔티티를 신설한다. 부분환불마다 1행 — payment_id, 환불 금액, PG 취소 거래 ID, refund_key(라인 멱등 키, 유니크), 생성 시각. refund_key 유니크가 DB 수준 이중 환불 방어를 부수로 제공한다. 전액 취소는 기존 `pgCancelTransactionId` 컬럼 경로를 유지한다(기존 경로 무변경).
- 문서: REQUIREMENTS 결제 절·DOMAIN_MODEL 결제 절에 부분환불 거래 기록을 반영한다.

### 5. 인증 표면표 압축

- 현상: REQUIREMENTS 인증 절의 엔드포인트 표면 분류가 4개 상세 표(60여 행)로 전수 인벤토리를 소유한다 — 엔드포인트 추가마다 갱신이 필요한 드리프트 재발 지점.
- 수정: 분류 규칙 4행(공개/본인/관리자/웹훅 — 각 분류의 강제 방식)과 분류당 대표 예시 소수로 압축한다. 전수 인벤토리 소유는 swagger-ui(OpenAPI 표면)에 명시 위임한다.
- 문서만 변경. 코드 무변경.

## 공통 원칙

- 항목 1~4는 TDD — 결함을 재현하는 실패 테스트를 먼저 쓰고 통과시킨다.
- 행동 변경은 같은 항목에서 REQUIREMENTS·DOMAIN_MODEL 해당 서술 갱신을 동반한다(문서 무모순 유지).
- 항목당 커밋 분리.
- 빌드 게이트(Spotless·NullAway·Error Prone·ArchUnit) 통과. 마이그레이션은 Flyway 신규 버전 파일로만 추가한다(기존 파일 불변).

## 완료 기준

- 항목 1~4: 결함 재현 테스트가 수정 후 통과하고 전체 빌드가 성공한다.
- 항목 5: REQUIREMENTS 인증 절이 규칙+예시 구조로 축소되고 전수 인벤토리 위임이 명시된다.
- 문서·코드 무모순: 변경된 행동의 문서 서술이 코드와 일치한다.
