# WORKPLAN — 차수 인덱스

커머스 API 확장 작업의 차수별 계획 인덱스다. 각 차수는 자기 파일을 진행 상태 저장소로 사용하며, 랄프 루프는 차수 파일 하나를 대상으로 돈다.

| 차수 | 파일 | 주제 | 상태 |
|---|---|---|---|
| 1차 | (완료 — 이력은 git과 `worklog/01~07` 참조) | 기능 공백 메우기: 쿠폰 목록, 비밀번호 변경, 체크아웃 미리보기, 바로구매, 선택구매, 배송지 관리, 상품 검색·정렬 | 완료 (`e8a7bd4`..`61f8e12`) |
| 2차 | [WORKPLAN-2.md](WORKPLAN-2.md) | 기능 확장: 토큰 갱신·로그아웃, 체크아웃 멱등키(Redis), 위시리스트, 상품 리뷰, 상품 문의 | 대기 |
| 3차 | [WORKPLAN-3.md](WORKPLAN-3.md) | 도메인 심화: 교환·반품, 비밀번호 재설정·이메일 인증, 상품 이미지, 카테고리 | 대기 |
| 4차 | [WORKPLAN-4.md](WORKPLAN-4.md) | 아키텍처 정렬·인프라: app-batch 분리 → app-admin 분리 → 트랜잭션 아웃박스 (순서 고정) | 대기 |

## 차수 운영 규칙

- 차수는 순서대로 진행한다. 2차·3차의 admin 표면이 4차에서 app-admin으로 이전되므로 4차를 앞당기지 않는다.
- 한 차수가 끝나면 main 머지 후 다음 차수를 시작한다.
- 각 차수의 진행 방법·작업 정의는 해당 차수 파일이 소유한다.
- 완료된 차수는 이 표의 상태만 갱신하고 파일은 남긴다.
